## Story

**As a** CEO/CMO de operação de Direct Response,
**I want** registrar e visualizar o P&L diário por produto,
**so that** eu veja quanto cada produto gastou, faturou e lucrou em qualquer período — tomando decisões de alocação de capital com base em dados reais.

```typescript

ad_performance_id: string  // PK
date: string               // DATE
platform: string           // ex: 'Facebook', 'Google'
investment: number         // spend
revenue: number            // receita
sales: number              // vendas
roas: number | null        // ROAS (já existe, calculado ou manual)
cpa: number | null         // custo por aquisição

notes: string | null
created_at: string
updated_at: string

// NOVOS CAMPOS (adicionados nesta story)
product: string | null     // nullable para não quebrar registros existentes
profit: number             // GENERATED ALWAYS AS (revenue - investment) STORED
```

**ATENÇÃO:** O campo `profit` é gerado automaticamente pelo banco. Não incluir no INSERT — o banco calcula automaticamente.
